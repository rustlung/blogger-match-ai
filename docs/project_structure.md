# Структура проекта

Проект организован как обычное Python-приложение с отдельными слоями для доменных моделей, сервисов, интеграций и тестов.

## Основные директории

`src/` — основной код приложения.

`src/models/` — Pydantic-модели и enum-контракты. Здесь находятся модели блогера, идеального профиля, результата матчинга, персонализированного предложения и batch-результатов.

`src/services/` — прикладная логика. Сервисы координируют чтение данных, внешние API, prompt building, LLM structured output, поиск, матчинг, предложения и экспорт.

`src/integrations/` — wrappers для сторонних SDK. Сейчас используется для Google Sheets.

`src/utils/` — общие утилиты: логгер и атомарная запись JSON.

`src/pipeline/` — зарезервированный слой pipeline. Текущая orchestration-логика находится в `src/main.py`.

`tests/` — unit-тесты для моделей, сервисов, интеграций и main-flow. Внешние API в тестах мокируются.

`data/` — локальная директория для входных данных, если они понадобятся вручную. Содержимое игнорируется git.

`results/` — runtime-артефакты выполнения. Содержимое игнорируется git, кроме `.gitkeep`.

`docs/` — документация проекта.

## Файлы в корне

`README.md` — краткая точка входа.

`.env.example` — шаблон переменных окружения без секретов.

`requirements.txt` — зависимости проекта.
